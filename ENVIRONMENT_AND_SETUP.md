## Environment and Setup

We evaluate our method in the Meta-World robotic manipulation benchmark, using the `button-press-topdown` task as the primary downstream control setting in the current experiments. In this task, a robotic arm must approach a button from above and press it successfully. This setting is useful for studying multiview visual representation learning because task success depends on both fine local geometry near the gripper and broader scene context that is often easier to perceive from external viewpoints.

Our visual input consists of three RGB camera views captured at each timestep. The first view is a gripper-mounted egocentric camera (`gripperPOV`), which provides a close-up first-person perspective aligned with the robot end effector. The other two views are third-person corner cameras (`corner2` and `corner3`), which provide more global views of the workspace, the tabletop, and the button location. In the policy training code, these are referred to by the aliases `first`, `third1`, and `third2`, respectively. This three-view setup is important because it creates a realistic multiview regime in which different cameras provide complementary information and no single view is guaranteed to be sufficient in all situations.

During downstream policy training, the agent is trained with the full three-camera observation set. However, evaluation is deliberately more demanding. We test the learned policy under four observation conditions: `first` only, `third1` only, `third2` only, and the full multiview setting with all three cameras available. This evaluation protocol measures not only standard task performance under the training-time observation regime, but also robustness to camera drop-off at test time. In other words, the single-camera evaluation settings probe whether the learned representation and policy can still support control when only one of the three views is available.

All images are resized to `84 x 84`, and observations are processed with a frame stack of `3`, so the agent receives short-term temporal context in addition to the instantaneous multiview image input. For the current Meta-World runs, the experiments use the `ours` backend configuration, train for up to `500,000` environment steps by default, begin learning after `2,000` seed steps, and evaluate every `10,000` steps over `20` evaluation episodes per camera condition. This setup gives a consistent basis for comparing both visual pretraining strategies and downstream policy-learning variants.

The overall pipeline is split into two stages. First, we pretrain a multiview ViT-based visual encoder using unlabeled image observations and objectives based on masked reconstruction and latent alignment across camera subsets. Second, we use that pretrained encoder in downstream reinforcement learning on Meta-World. This separation is central to the paper: the pretraining stage is meant to learn a shared latent space in which partial camera observations remain compatible with the full multiview latent, while the policy-training stage tests whether that representation actually improves control robustness under reduced-view deployment.

This environment setup is especially relevant to our hypothesis. The first-person gripper camera tends to provide strong local information about contact and button alignment, whereas the third-person cameras provide better global context about object position and scene layout. A robust multiview representation should therefore capture information that transfers across these observation modes, allowing a policy trained with full camera access to remain effective even when only one camera is available at evaluation time. The experiments that follow test exactly this claim by comparing both visual pretraining methods and downstream policy-training variants under these controlled single-view and multiview evaluation settings.
